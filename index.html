このシステムで学ぶプログラミングの核心

1. プログラムの基本三構造「順次処理」

あなたが「命令の追加」ボタンを押すと、右側のリストに上から順番に命令が積み重なります。「プログラムを実行」ボタンを押したとき、コンピュータは内部の配列から命令を一つずつ取り出し、上の行から下の行へ向かって順番に実行していきます。これをプログラミングの基本構造の一つである順次処理と呼びます。

2. データの保持と「変数」

ロボットが今どこにいて、どちらを向いているか、コンピュータは「変数」という箱に記憶しています。このプログラムの裏側では、以下のようなオブジェクト（変数の塊）が常に書き換えられています。

let robot = {
    x: 0,    // ロボットの横の位置（0から4）
    y: 0,    // ロボットの縦の位置（0から4）
    dir: 0   // ロボットの向き（0:右、1:下、2:左、3:上）
};

「右を向く」という命令が実行されると、内部では robot.dir = (robot.dir + 1) % 4 という計算が行われ、変数の中身が書き換わります（代入）。画面の表示が切り替わるのは、コンピュータがこの変数の中身を読み取って、一瞬で画面を描き直しているからです。

3. 進入禁止を判断する「条件分岐」

ロボットが前進しようとしたとき、目の前が壁（灰色のマス）だったり、画面の外だったりした場合は進めません。これを制御しているのが条件分岐（if文）です。

if (次の位置がマップの範囲内である && 次の位置が壁ではない) {
    実際に座標を書き換えて前進する;
} else {
    前進せずにその場にとどまる（エラーを防ぐ）;
}

この「もし〜なら処理する、そうでなければ処理しない」というロジックは、Pythonなどのテキストプログラミングでも全く同じ考え方で記述します。
